## Why

当前系统缺少独立的后台授权入口，无法让管理员以低成本给指定账号发放模型调用额度，导致权限分发和配额调整需要人工介入代码或数据库。现在需要补充一个与现有界面风格一致的后台管理页面，以支持管理员快速、安全地完成账号授权操作。

## What Changes

- 新增后台管理页面入口与页面容器，复用现有 UI 组件和视觉样式（输入框、按钮、卡片、动画风格等），保持与当前产品一致的设计语言。
- 新增管理员访问保护流程：进入后台管理页前需先验证管理员密码（默认密码为 `liao123...`）。
- 新增授权表单能力：管理员可输入“用户名”和“调用大模型次数”，其中调用次数默认值为 10。
- 新增授权提交流程：完成输入后可提交并保存该用户的模型调用额度（用于后续实际权限生效）。
- 新增基础校验与反馈：对管理员密码、用户名、调用次数做必填与格式校验，并在提交后展示成功/失败反馈。

## Capabilities

### New Capabilities

- `admin-access-gate`: 提供后台管理页的管理员密码校验与受控进入流程。
- `admin-user-quota-grant`: 提供“用户名 + 调用次数”授权表单、默认值、校验与提交流程。

### Modified Capabilities

- 无

## Impact

- 受影响代码范围：`App.tsx` 页面状态切换逻辑、侧边栏导航、`components/` 下新增后台管理页组件与表单逻辑。
- 受影响交互：新增后台入口和管理员登录闸口，扩展现有用户操作路径。
- API/数据影响：需要定义或复用“按用户名设置调用次数”的后端接口（若暂未接入则需先使用前端占位流程）。
- 依赖影响：不新增外部 UI 依赖，优先迁移现有组件和样式体系。
